Web Server Install

  • Task: Install the NGINX Web server via the terminal/shell and take screenshots of your work.
  • Purpose: Demonstrate your ability to install and set up a web Server via the terminal/shell.

Instructions

Part 1: Determine whether you will run a web server on your local machine or in Cloud

We recommend you install the webserver in the AWS system, but in case you have problems, the local option is available. Decide which one, AWS or local VM in Virtual Box (or UTM for some Mac/Apple).

AWS (recommended)

Log into https://awsacademy.instructure.com/, go to "Modules", click "Learner Lab", start the learner lab, click the green AWS button, make sure you are in the correct region, search for EC2, find your instance, click "Instance State" > "Start instance", open your https://{your instance IP address}:8443. Login to your DC Viewer with your username and password.

Local VM

Open Virtual Box or UTM and start your Linux VM.

Part 2: Open the terminal/shell to run the install commands.

  • Open the terminal/shell and type this code to get latest update information for packages/apps/security:
    • sudo apt update
  • Then type this code to install teh NGINX webserver:
    • sudo apt install nginx -y

Part 3: View your new website installed with your NGINX webserver.

  • Open Firefox inside of AWS or inside your local VM (not on your laptop).
  • Go to the URL (this loads the local server/computer webpage): http://localhost/
  • Make sure not to put https, it is http without the 's'.
  • You should see the NGINX welcome page.
  • If that doesn't work, try http://127.0.0.1/ instead.

Part 4: Modify the default webpage to have your name on it

  • Modify the index file that was created when you installed NGINX in /var/www/html.
  • In the VM or AWS terminal, type:
    • sudo gedit /var/www/html/index.html
  • Type your name in the file so it shows on the Firefox browser page you opened above. Put your name and save it. Click refresh on the browser.
  • Alternatively, you can have AI generated HTML/CSS/Javascript into one file to create your own custom webpage about your favorite hobby or a portfolio webpage about you.
  • Take a screenshot.

Part 5 (Optional But Really Cool): Make your webserver work outside your VM

  • Get the website working in the real-world not just on the localhost.
  • In AWS, go to your EC2 instance and select it, then under the security tab, click "Security Groups", click the security group link, click "Edit inbound rules", click "Add rule", select "HTTP" from the dropdown, click "Save rules".
  • For the inbound source rules (use CIDR 0.0.0.0/0).
  • Find your public IPv4 address of your instance in the AWS EC2 dashboard
    • Go to Firefox on your laptop (not in the VM or AWS)
    • Type your public IPv4 address into the URL bar (http://{your public IPv4 address}/ )
    • Make sure to not put the 's' in https.
    • You should see your webpage with your name on it.
    • Take a screenshot.

Extra Mile

On the AWS system, you could install or add a database server such as MYSQL, postgresql, dynamodb, or redis.
